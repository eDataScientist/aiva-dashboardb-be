AI Quality Metrics & Intent Classification Arabia Insurance AI Dashboard — Phase 2 Reference Guide Phase 2 evaluates every customer conversation daily using an AI Judge across four dimension groups. Each score is 1–10 with AI-generated reasoning. A fifth area — Intent Classification — identifies the customer's primary purpose. AI Performance How well the AI assistant handled the interaction. Metric What it measures Relevancy AI responses matched actual questions Accuracy Factual & procedural correctness Completeness AI fully addressed customer needs Clarity Clear, understandable communication Tone Appropriate for professional insurance context Conversation Health Structural quality and flow of the conversation. Metric Type What it measures Resolution Boolean Customer's need was fully met Repetition Score 1–10 Avoided unnecessary repetition (higher = better) Loop Detected Boolean Circular pattern with no progress occurred User Signals Customer experience inferred from conversation content. Metric Type What it measures Satisfaction 1–10 Inferred customer satisfaction level Frustration 1–10 Inferred frustration (higher = more frustrated) User Relevancy Boolean True = genuine; False = non-genuine/wasteful Escalation Whether and how a human handoff occurred. Metric Type Values Escalation Occurred Boolean True / False Escalation Type Enum Natural — planned; Failure — AI breakdown; None Highlights —  Auto-flagged if: Frustration ≥ 7, Escalation = Failure, Loop detected, Accuracy ≤ 3, Unresolved + Satisfaction ≤ 3, or User Relevancy = False. Intent Classification The AI Judge classifies each conversation's dominant customer intent against a fixed 16-label taxonomy. The label reflects what the customer spent the most effort on. Multi-intent conversations pick the strongest signal. Reasoning is always returned in English regardless of conversation language (Arabic, English, or mixed).
